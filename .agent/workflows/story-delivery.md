# Story Delivery

## Lead

`scrum-orchestrator` or the story's primary specialist

## Support

impacted developers, `qa-engineer`, optional `security-engineer`, optional `devops-engineer`

## Steps

1. start from the story packet
2. implement within the agreed scope
3. run verification and capture evidence
4. review docs impact and release implications
5. confirm done or return with blockers

## Deliverables

- implementation summary
- verification evidence
- done or blocked decision
