# Release Readiness

## Lead

`scrum-master`

## Support

`qa-engineer`, `security-engineer`, `devops-engineer`, `product-owner`

## Steps

1. review blocking findings and open risks
2. confirm verification evidence and rollback plan
3. confirm release notes and stakeholder expectations
4. make explicit ship or no-ship decision

## Deliverables

- release decision
- blocker list
- rollback plan
- release notes
