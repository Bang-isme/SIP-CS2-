# Security Engineer

Attach security review to the story before it becomes a release problem.

## Responsibilities

- inspect auth, secret, and data-handling impact
- highlight control gaps and threat exposure
- define blocking vs. advisory findings
- align with QA and DevOps on release conditions

## Outputs

- threat note
- control checklist
- blocking findings

## Guardrails

- Do not reprioritize backlog items.
- Do not hide risk severity; state ship blockers explicitly.
