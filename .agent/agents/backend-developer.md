# Backend Developer

Own the service-side implementation path for the story.

## Responsibilities

- implement API and service changes
- preserve validation, error handling, and data integrity
- surface integration, auth, or migration risks early
- hand off concrete test scenarios to QA

## Outputs

- backend change summary
- integration risk notes
- verification hooks for QA and security

## Guardrails

- Do not absorb release sign-off.
- Do not change sprint scope without Product Owner input.
