# Session Summary

- Date: 2026-04-22
- Focus: Add a repeatable and resumable Case 2-4 demo evidence pack

## What changed

### Added a one-command evidence bundle builder

- Added `scripts/build-demo-evidence-pack.mjs`
- Added `npm run demo:evidence:build`
- The builder now orchestrates:
  - `verify:backend`
  - `verify:frontend`
  - `verify:case3`
  - `verify:case4:operations-demo`
  - `demo:dashboard:prepare`
  - optional screenshot capture
- Added canonical bundle planning in `src/utils/demoEvidenceCapturePlan.js`

### Captured structured demo-prep output

- Updated `scripts/prepare-dashboard-demo.js`
- It now supports `DASHBOARD_DEMO_PREP_OUTPUT_PATH`
- This lets the evidence builder persist the current demo-prep report directly into the evidence bundle

### Hardened the builder for local Windows runtime and interrupted runs

- Fixed Windows process spawning by routing npm commands through:
  - `cmd.exe /d /s /c npm ...`
- Added resume-oriented flags:
  - `DEMO_EVIDENCE_SKIP_VERIFY_BACKEND`
  - `DEMO_EVIDENCE_SKIP_VERIFY_FRONTEND`
  - `DEMO_EVIDENCE_SKIP_VERIFY_CASE3`
  - `DEMO_EVIDENCE_SKIP_VERIFY_CASE4`
  - `DEMO_EVIDENCE_USE_RUNNING_STACK`
  - `DEMO_EVIDENCE_SKIP_CAPTURE`
- This allows an interrupted build to reuse existing logs and an already-running stack instead of redoing the full proof chain

### Updated docs for the new proof path

- Updated `docs/case1_4_operational_uplift_plan.md`
- Updated `docs/demo_pass_checklist.md`
